# PRD — Byredo Interaction Commerce Website

## 1. 서비스 목적
- **브랜드 경험의 디지털화**: 바이레도(Byredo) 특유의 미니멀리즘과 예술적 세계관을 웹 인터랙션으로 구현.
- **몰입형 탐색**: 단순한 리스트 나열이 아닌, 스크롤과 3D 오브젝트를 활용한 감각적인 제품 탐색 경험 제공.

## 2. 타겟 사용자
- 바이레도 브랜드를 선호하거나 감도 높은 브랜드 경험을 중시하는 **20~40대 얼리어답터 및 패션/뷰티 관여도가 높은 사용자**.

## 3. 핵심 기능 (Core Features)
- **F1. 브랜드 중심 제품 탐색**: 스크롤 기반 인터랙션 및 컬렉션 단위의 내러티브 중심 구성.
- **F2. 3D/비주얼 중심 제품 상세**: Three.js 또는 WebGL을 활용한 3D 오브젝트 인터랙션 및 텍스트 최소화.
- **F3. 기본 커머스 액션**: 장바구니(Cart) 담기 및 관심 상품(Wishlist) 등록 기능.

## 4. IA (Information Architecture) / 페이지 목록
- **Home**: 브랜드 무드 전달 및 탐색 진입점.
- **Products**: 전체 상품 리스트 (4x4 그리드).
- **Product Detail**: 3D 인터랙션 및 상세 정보, 구매 액션.

---

## 5. 페이지별 요구사항

### 5.1 Home (메인 페이지)
**목표**: 첫 진입 시 브랜드 아이덴티티 전달 및 자연스러운 탐색 유도.

- **구성 섹션**
  1. 하프앤하프(Half & Half) 캐러셀 비주얼 섹션
  2. 상품 썸네일 4×4 그리드
  3. 뉴스레터 구독 섹션
  4. 메인 CTA 버튼 영역

- **필수 UI 요소**
  - **하프앤하프 캐러셀**: 좌/우 분할 레이아웃, 네비게이션(이전/다음), 스크롤 유도 아이콘.
  - **4×4 그리드**: 직사각형 썸네일, 최소한의 상품명, 상세 페이지 연결.
  - **뉴스레터**: 이메일 Input, 구독 Button, 유효성 체크 문구.
  - **CTA 버튼**: `제품 보러가기`, `컬렉션 보기`.

- **상태 및 예외 처리**
  - 로딩 중: 스켈레톤 UI 적용.
  - 이메일 형식 오류: 인풋 하단 에러 메시지 노출.

---

### 5.2 Products (상품 리스트 페이지)
**목표**: 전체 제품 라인업을 시각적으로 확인하고 원하는 상세 페이지로 빠르게 진입.

- **구성 섹션**
  1. 4×4 상품 그리드
  2. (선택) 컬렉션 필터 및 정렬 드롭다운

- **필수 UI 요소**
  - **그리드 레이아웃**: 고해상도 이미지 기반의 카드 UI.
  - **상품 정보**: 상품명, 가격(선택사항).
  - **인터랙션**: 카드 호버(Hover) 시 이미지 확대 또는 세컨드 이미지 노출.

- **상태 및 예외 처리**
  - 이미지 로딩 실패: 대체 플레이스홀더 이미지 노출.
  - 결과 없음: "검색 결과가 없습니다" 안내 문구.

---

### 5.3 Product Detail (상품 상세 페이지 — 3D)
**목표**: 3D 모델링을 통해 제품의 질감과 형태를 입체적으로 경험하고 구매 의사 결정.

- **구성 섹션**
  1. 3D 인터랙션 뷰어 (Main)
  2. 제품 정보 (Title, Description, Price)
  3. 액션 버튼 (Add to Cart, Wishlist)
  4. 보조 정보 (Fragrance Notes, Brand Story)

- **필수 UI 요소**
  - **3D 뷰어**: 360도 회전(Drag), 줌인/아웃 지원.
  - **사용자 힌트**: "Drag to rotate" 등의 안내 문구.
  - **Commerce 버튼**: `장바구니 담기`, `찜하기(Heart Icon)` 토글.
  - **향 노트**: 탑/하트/베이스 노트 텍스트 리스트.

- **상태 및 예외 처리**
  - **3D 로딩 실패**: 고해상도 정적 이미지(2D)로 대체 및 Fallback UI 제공.
  - **액션 성공**: 토스트(Toast) 알림으로 피드백 제공.

---

## 6. 사용자 흐름 (User Flow)
1. **Home 진입** → 브랜드 캐러셀 확인 및 스크롤 탐색.
2. **Products 이동** → 4x4 그리드에서 상품 스캔.
3. **특정 상품 클릭** → 상세 페이지 진입.
4. **3D 인터랙션** → 제품 회전 및 디테일 확인.
5. **액션 수행** → 장바구니 담기 또는 찜하기 클릭.

---

## 7. 비기능 요구사항
- **반응형 디자인**: Mobile, Tablet, Desktop 최적화.
- **성능**: 3D 에셋 최적화(GLB/GLTF 용량 관리) 및 지연 로딩(Lazy Loading).
- **접근성**: 텍스트 대비 확보 및 3D 조작 가이드 제공.

## 8. 향후 로드맵 (Phase 2)
- 회원가입 및 로그인 시스템.
- 실제 결제 게이트웨이(PG) 연동.
- 향 맞춤 추천(Quiz) 서비스.
- 관리자(CMS) 페이지 개발.

## 9. MVP 완료 기준 (Acceptance Criteria)
- [ ] Home에서 Product Detail까지의 탐색 경로가 막힘없이 작동하는가?
- [ ] 상세 페이지의 3D 모델이 부드럽게 회전 및 제어되는가?
- [ ] 3D 로딩 실패 시 Fallback 이미지가 정상적으로 출력되는가?
- [ ] 장바구니/찜하기 클릭 시 상태 변화가 즉시 반영되는가?
